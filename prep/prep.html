  • System troubleshooting
  • High CPU usage :

top: To get CPU metrics type ‘1’ after running top command.

uptime: Uptime provides information about how long the system is up, the number of active users, and the load averages in the span of 1 minute, 5 minutes and 15 minutes. The last three numbers help you understand if the usage spike is long term or short term. The decimal represents the number of active tasks requesting CPU resources to perform an action. If the last number is too high, then that is a problem that should be taken care of. If you have 1 core, having a value greater than 1(1.5), the difference(0.5) represents the percentage of the number of processes that are waiting in line to be executed.

Check file /proc/cpuinfo to get the number of processors.

nice: command to limit the CPU resources, By default a new process starts with a nice value of 0. nice values should lie between -20 and 19.
